# Generic Evolve Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the agentfolio scaffold work for any single-project repo by replacing hardcoded agentfolio references with config-driven behavior, gated by an interactive onboarding step.

**Architecture:** A config file (`state/evolve_config.md`) generated during interactive CLI onboarding drives all workflow behavior. Every workflow checks for this file immediately after checkout — no file means no execution. The config is injected into each workflow's prompt so Claude adapts behavior (LLM-level adaptation, not bash-level branching).

**Tech Stack:** GitHub Actions YAML, bash, Claude Code CLI (`claude -p`), Astro (profile page)

**Spec:** `docs/superpowers/specs/2026-03-22-generic-evolve-design.md`

## Phase 3: Interactive Onboarding

### Task 5: Rewrite onboarding.md as interactive CLI guide

The onboarding doc becomes instructions that Claude follows during a CLI session. It must analyze the repo, present fit estimates, ask questions, and write the config.

**Files:**
- Modify: `docs/onboarding.md`

- [ ] **Step 1: Rewrite onboarding.md**

The doc should instruct Claude to:

1. Copy workflow files from agentfolio (same as current)
2. Analyze the repo:
   - Read package.json / requirements.txt / go.mod / Cargo.toml / etc.
   - Read existing CLAUDE.md or README.md
   - List file extensions to detect language mix
   - Check for Dockerfile, deploy configs, CI configs
   - Check for test frameworks (jest, pytest, go test, etc.)
3. Present findings to the user:
   - "I detected: Python 3.x, FastAPI, PostgreSQL (from requirements.txt)"
   - "Build: no build step detected. Test: pytest found."
   - Show workflow fit table with HIGH/MEDIUM/SKIP for each
4. Ask questions one at a time:
   - "How do you deploy this project?"
   - "Is this a public or private repo?"
   - "These workflows will run autonomously with full permissions. OK to proceed?"
5. Auto-discover research sources:
   - Map major dependencies to GitHub repos
   - Always include verkyyi/agentfolio
   - Present list: "I'll watch these for updates: [list]. Want to add or remove any?"
6. Write `state/evolve_config.md` with confirmed answers
7. Initialize state files (project_state.md, agent_log.md, etc.)
8. Commit and push
