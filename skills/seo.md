# SEO & Discoverability

## What We Optimize For
Two audiences: human search engines (Google) and AI systems
(ChatGPT, Claude, Perplexity). Tactics differ slightly.

## Structured Data (JSON-LD)
Maintained in src/layouts/Base.astro. Update when profile changes.

### Person Schema (required)
```json
{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "[name from profile.json]",
  "url": "[site_url]",
  "description": "[bio]",
  "sameAs": ["[github]", "[blog]"],
  "knowsAbout": ["[skill1]", "[skill2]"]
}
```

### ItemList for Projects (add when > 3 projects)
```json
{
  "@context": "https://schema.org",
  "@type": "ItemList",
  "name": "Projects by [name]",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "[project title]",
      "url": "[project url or github]",
      "description": "[project description]"
    }
  ]
}
```

## Meta Tags Checklist
Every page must have:
- [ ] <title> — unique per page, includes person name
- [ ] <meta name="description"> — 120-160 chars
- [ ] <meta property="og:title">
- [ ] <meta property="og:description">
- [ ] <meta property="og:type"> — "website" for index, "article" for posts
- [ ] <link rel="canonical">

## llms.txt Format
AI-readable plain text at /llms.txt. Keep it factual and unambiguous.
AI systems cite pages that are authoritative about their subject.
```
Name: [Full Name]
Role: [Role / Title]
Location: [City, Country]
Status: [open | busy | closed]

Building:
- [Project 1]: [one sentence]
- [Project 2]: [one sentence]

Expertise:
- [skill or technology]
- [skill or technology]

Contact: [site_url]/contact (or feedback link)
Profile JSON: [site_url]/api/profile.json
GitHub: [github url]
```

Update llms.txt whenever:
- A new featured project is added
- Availability status changes
- Skills list changes significantly

## Sitemap
Generated by @astrojs/sitemap integration automatically on build.
No manual maintenance needed unless pages are excluded.

## Robots.txt
Located at public/robots.txt. Key rule: explicitly allow AI crawlers.
```
User-agent: GPTBot
Allow: /

User-agent: Claude-Web
Allow: /
```

## Lighthouse Targets
- Performance: >= 90
- Accessibility: >= 95
- Best Practices: >= 90
- SEO: >= 95

Common fixes for Lighthouse performance:
- Image optimization: use WebP, add width/height attributes
- Remove unused CSS: Tailwind purge should handle this
- Defer non-critical JS: Astro handles most of this natively
- Add explicit font-display: swap for web fonts

## AI Discoverability Signals
In order of impact:
1. llms.txt — direct AI-readable file
2. /api/profile.json — structured data endpoint
3. JSON-LD structured data on pages
4. Consistent naming across pages (same name, same role)
5. Unambiguous factual statements (not marketing language)
6. Fresh content signals (updated timestamps in structured data)

## Gotchas
- Lighthouse CI in growth.yml requires the live Pages site — not localhost
- Schema markup validator: https://validator.schema.org
- AI crawlers don't execute JavaScript — structured data in <head> only
- robots.txt is at /robots.txt (public/ folder in Astro)
- Sitemap at /sitemap-index.xml (generated by astrojs/sitemap)
